# 单例模式

**类只有一个实例**

---
**使用场景**
1. 要求生成唯一序列号
2. 共享访问点或共享资源
3. 创建一个对象消耗的资源过多
4. 需要定义大量的静态常量和静态方法


---
**优点**
1. 减少内存开支
2. 减少性能开销
3. 避免对资源的多重占用
4. 可以在系统设置全局访问点，优化和共享资源访问

---
**缺点**
1. 一般没有接口，扩展难
2. 与单一职责原则冲突

---

1. [懒汉式](./way1)
   1. 缺点：不管是不是用都会创建对象
